Implementación de OAuth2 en Microservicios

OAuth2 es un protocolo de autorización que permite a las aplicaciones obtener acceso limitado a recursos de usuario en un servidor sin necesidad de exponer sus credenciales. Es ampliamente utilizado para garantizar la seguridad en aplicaciones distribuidas y microservicios.

Arquitectura de Microservicios

En nuestro proyecto, hemos implementado dos microservicios principales:

  • msvc-cursos: Este microservicio gestiona toda la lógica relacionada con los cursos.
  • msvc-usuarios: Este microservicio gestiona la información de los usuarios, incluyendo autenticación y autorización.

Flujo de OAuth2

  1. Cliente: El usuario inicia sesión en la aplicación (cliente) y solicita acceso a un recurso protegido.
  2. Autorización: La aplicación redirige al usuario a una página de autorización, donde puede otorgar permisos a la aplicación.
  3. Token de Acceso: Una vez autorizado, el cliente recibe un token de acceso que utiliza para solicitar recursos protegidos de los microservicios.
  4. Acceso a Recursos: El cliente utiliza el token de acceso para interactuar con los microservicios msvc-cursos y msvc-usuarios, permitiendo acceder solo a la información autorizada.
